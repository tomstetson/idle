# Alpha v1.0 Full Design

**Date**: 2026-03-08
**Status**: Approved
**Goal**: Clean backlog, validate security, ship stable CLI + TestFlight build

## Strategy

Security-first, then tests, then bug fixes, then ship. Instead of adding Cloudflare Access complexity, we validate the existing security posture (device-key crypto auth + E2E encryption + WAF). If it holds, the web app goes public — simpler architecture, fewer moving parts.

PWA auth persistence is deferred (Phase 4 backlog) — the native mobile app is free and the primary platform. PWA was an Idle fork addition; upstream Happy doesn't have it.

GitHub OAuth is disabled cleanly — upstream Happy uses it only for optional profile info, not authentication. Remove the broken UI, keep server endpoints intact, defer to Phase 4.

## Phase A: Security Audit

10-area review producing `docs/security/alpha-security-audit.md`:

1. **API attack surface** — Enumerate all routes, test unauthenticated access
2. **Auth boundary** — Verify nothing works without signed keypair. Replayed/expired/malformed tokens
3. **Credential exposure** — GitHub OAuth tokens in PGlite: encrypted or plaintext? What leaks if DB dumped?
4. **E2E encryption** — Server never sees plaintext session content. Key derivation, nonce reuse check
5. **Web app vectors** — XSS (user content in chat), CSRF, open redirects, CSP headers
6. **Rate limiting** — Auth endpoints, WebSocket connections, API abuse resistance
7. **Infrastructure** — VPS SSH config, nginx headers, exposed ports, Cloudflare bypass via direct IP
8. **Data at rest** — What's in PGlite plaintext? Session names, machine info, user metadata
9. **Dependency/supply chain** — `npm audit`, outdated deps, known CVEs, lockfile integrity
10. **Logging review** — No tokens, keys, session content, or PII in server logs, CLI file logs, nginx access logs

**Output**: Findings doc with risk ratings (Critical/High/Medium/Low) + ADR for public access decision.

### Testing principles
- Test behavior, not implementation
- No mocking (upstream convention) — real API calls where possible
- Descriptive names: `test_rejects_expired_handoff_token`
- Security edge cases: malformed input, boundary values, permission violations

- **Files**: `OAuthView.tsx`, settings UI

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| No Cloudflare Access | Device-key crypto is the real auth boundary. CF Access adds complexity for marginal benefit. Validate via security audit. |
| No email collection | Matches upstream Happy model. Users identified by public keys. |
| PWA auth persistence deferred | Native app is primary. PWA was Idle fork addition, not upstream. |
| GitHub OAuth disabled, not fixed | Upstream uses it only for optional profile info. Broken UI is worse than no UI. |
| Tests before bug fixes | Regression safety net for all fix work. Security findings feed test cases. |
| Public web app (pending audit) | If audit passes, skip CF Access entirely. If critical findings, re-evaluate. |
